문서

LLM Quality Radar의 완전한 가이드입니다. 설치부터 고급 사용법까지 모든 것을 단계별로 설명합니다.

개요

LLM Quality Radar는 Google ADK(Application Development Kit)와 Playwright MCP(Model Context Protocol)를 연계한 혁신적인 웹 자동화 테스트 및 AI 기반 품질 분석 시스템입니다.

AI 기반 분석

Gemini 2.0 Flash 모델을 활용한 지능형 웹페이지 품질 분석

자동화 테스트

Playwright MCP 기반의 안정적이고 빠른 웹 자동화 테스트

실시간 모니터링

웹 애플리케이션의 성능과 품질을 실시간으로 모니터링

핵심 가치: 복잡한 스크립트 작성 없이 자연어로 테스트 시나리오를 작성하고, AI가 자동으로 최적화된 테스트 코드를 생성합니다.

시작하기

설치

LLM Quality Radar를 설치하는 방법은 여러 가지가 있습니다.

1. 소스 코드에서 설치

# 저장소 클론
git clone https://github.com/your-org/llm-quality-radar.git
cd llm-quality-radar

# Python 의존성 설치
pip install -r requirements.txt

# Node.js 의존성 설치
npm install

2. Docker로 설치

# Docker 이미지 빌드
docker build -t llm-quality-radar .

# 컨테이너 실행
docker run -p 8080:8080 -p 3001:3001 llm-quality-radar

빠른 시작

환경 변수 설정

Google ADK API 키와 기타 필요한 환경 변수를 설정합니다.

# .env 파일 생성
GOOGLE_ADK_API_KEY=your_api_key_here
FIREBASE_PROJECT_ID=your_project_id
MCP_SERVER_PORT=3001

MCP 서버 시작

Playwright MCP 서버를 백그라운드에서 실행합니다.

# MCP 서버 시작
node servers/playwright_mcp_server.js

메인 애플리케이션 실행

웹 서버와 API를 시작합니다.

# 웹 서버 시작
python apps/web_server.py

# 또는 데모 시스템 실행
python demos/run_system.py

웹 인터페이스 접속

브라우저에서 http://localhost:8080에 접속합니다.

설치가 완료되면 테스트 생성기 페이지에서 첫 번째 테스트를 만들어보세요!

주요 컴포넌트

  • Google ADK Integration: Gemini 2.0 Flash 모델을 활용한 AI 기반 분석
  • Playwright MCP Client: 웹 브라우저 자동화 및 테스트 실행
  • Quality Monitor: 실시간 품질 메트릭 수집 및 분석
  • Auto Healing System: ML 기반 자동 복구 및 최적화
  • Firebase Integration: 클라우드 기반 데이터 저장 및 배포

주요 기능

AI 기반 품질 분석

Google ADK의 Gemini 2.0 Flash 모델을 활용하여 웹페이지의 품질을 종합적으로 분석합니다.

분석 항목: 성능, 접근성, SEO, 사용자 경험, 보안, 모바일 최적화

# 품질 분석 실행 예시
from core.quality_monitor import QualityMonitor

monitor = QualityMonitor()
result = await monitor.analyze_page("https://example.com")

print(f"품질 점수: {result.quality_score}/100")
print(f"권장사항: {result.recommendations}")

웹 자동화 테스트

Playwright MCP를 통해 다양한 브라우저에서 안정적인 웹 자동화 테스트를 수행합니다.

  • ✅ 크로스 브라우저 지원 (Chrome, Firefox, Safari)
  • ✅ 헤드리스 및 헤드풀 모드
  • ✅ 모바일 디바이스 에뮬레이션
  • ✅ 스크린샷 및 비디오 캡처
  • ✅ 네트워크 가로채기 및 분석

실시간 모니터링

웹 애플리케이션의 상태를 실시간으로 모니터링하고 문제점을 즉시 감지합니다.

# 실시간 모니터링 설정
from core.operational_manager import OperationalManager

manager = OperationalManager()
await manager.start_monitoring([
    "https://your-site.com",
    "https://api.your-site.com/health"
])

환경 설정

LLM Quality Radar의 다양한 설정 옵션을 구성하는 방법입니다.

환경 변수

# .env 파일 예시
# Google ADK 설정
GOOGLE_ADK_API_KEY=your_gemini_api_key
GOOGLE_ADK_MODEL=gemini-2.0-flash
GOOGLE_ADK_TEMPERATURE=0.1

# Firebase 설정
FIREBASE_PROJECT_ID=llm-quality-radar-2e61a
FIREBASE_API_KEY=your_firebase_api_key

# MCP 서버 설정
MCP_SERVER_HOST=localhost
MCP_SERVER_PORT=3001
MCP_PROTOCOL=http

# 테스트 설정
DEFAULT_TIMEOUT=30000
HEADLESS_MODE=true
SLOW_MO=0

# 로깅
LOG_LEVEL=info
LOG_FILE=quality_radar.log

설정 파일

config/adk_config.py에서 고급 설정을 관리할 수 있습니다.

문제 해결

자주 발생하는 문제

1. MCP 서버 연결 실패

증상: "Failed to connect to MCP server" 오류

해결책:

  • MCP 서버가 실행 중인지 확인: node servers/playwright_mcp_server.js
  • 포트 3001이 사용 가능한지 확인: netstat -an | grep 3001
  • 방화벽 설정 확인

2. Google ADK API 오류

증상: "Invalid API key" 또는 "Quota exceeded" 오류

해결책:

  • API 키가 올바르게 설정되어 있는지 확인
  • Google Cloud Console에서 할당량 확인
  • ADK API가 활성화되어 있는지 확인

로그 확인

# 애플리케이션 로그 확인
tail -f quality_radar.log

# MCP 서버 로그 확인
tail -f mcp_server.log

# Docker 컨테이너 로그 확인
docker logs quality-radar
