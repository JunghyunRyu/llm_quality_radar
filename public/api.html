API 문서

LLM Quality Radar의 RESTful API를 사용하여 웹 자동화 테스트를 프로그래밍 방식으로 제어하고 AI 기반 품질 분석 기능을 애플리케이션에 통합할 수 있습니다.

개요

Quality Radar API는 Google ADK와 Playwright MCP를 활용한 웹 테스트 자동화 서비스를 제공합니다. 모든 API 엔드포인트는 HTTPS를 통해 안전하게 제공되며, JSON 형식으로 데이터를 주고받습니다.

기본 URL

https://us-central1-llm-quality-radar-2e61a.cloudfunctions.net/

지원하는 기능

  • 🤖 AI 기반 테스트 코드 생성
  • 🎭 Playwright MCP 웹 자동화
  • 📊 실시간 품질 분석
  • 🔄 자동 복구 시스템
  • 📈 테스트 결과 저장 및 조회

인증

API 호출 시 인증이 필요한 엔드포인트에는 Authorization 헤더에 Bearer 토큰을 포함해야 합니다.

중요: API 키는 서버 사이드에서만 사용하고, 클라이언트 코드에 노출하지 마세요.

인증 헤더 예시

Authorization: Bearer YOUR_API_KEY_HERE
Content-Type: application/json

테스트 생성

POST
/generateTestCode

Google ADK를 활용하여 자연어 설명으로부터 Playwright 테스트 코드를 자동 생성합니다.

요청 파라미터

파라미터 타입 필수 설명
url string ✅ 테스트할 웹사이트 URL
description string ❌ 테스트에 대한 설명
test_scenarios array ✅ 테스트 시나리오 배열
auto_healing boolean ❌ 자동 복구 기능 활성화 (기본값: false)
quality_checks boolean ❌ 품질 검사 포함 (기본값: false)

요청 예시

{
  "url": "https://www.google.com",
  "description": "구글 검색 기능 테스트",
  "test_scenarios": [
    {
      "action": "wait",
      "selector": "input[name='q']",
      "description": "검색창 대기"
    },
    {
      "action": "type",
      "selector": "input[name='q']",
      "value": "LLM Quality Radar",
      "description": "검색어 입력"
    },
    {
      "action": "click",
      "selector": "input[type='submit']",
      "description": "검색 버튼 클릭"
    }
  ],
  "auto_healing": true,
  "quality_checks": true
}

응답 예시 (200 OK)

{
  "success": true,
  "test_code": "const { test, expect } = require('@playwright/test');\n\ntest('구글 검색 기능 테스트', async ({ page }) => {\n  await page.goto('https://www.google.com');\n  await page.waitForSelector('input[name=\"q\"]');\n  await page.fill('input[name=\"q\"]', 'LLM Quality Radar');\n  await page.click('input[type=\"submit\"]');\n});",
  "quality_score": 85.7,
  "auto_healing_enabled": true,
  "execution_time": "2.3s",
  "recommendations": [
    "검색 결과 페이지 로딩 검증 추가 권장",
    "모바일 뷰포트 테스트 고려"
  ]
}
POST
/generateAutoTestCases

URL을 분석하여 자동으로 포괄적인 테스트 케이스를 생성합니다.

요청 파라미터

파라미터 타입 필수 설명
url string ✅ 분석할 웹사이트 URL
test_type string ❌ 테스트 유형 ('comprehensive', 'basic', 'performance')

테스트 결과

POST
/saveTestResult

테스트 실행 결과를 Firestore에 저장합니다.

GET
/getTestResults

저장된 테스트 결과를 조회합니다.

쿼리 파라미터

파라미터 타입 필수 설명
limit integer ❌ 반환할 결과 수 (기본값: 50, 최대: 100)

시스템 모니터링

GET
/healthCheck

시스템 상태와 각 서비스의 가용성을 확인합니다.

응답 예시 (200 OK)

{
  "status": "healthy",
  "timestamp": "2024-01-15T09:30:00.000Z",
  "version": "1.0.0",
  "services": {
    "firebase": "connected",
    "adk": "available",
    "mcp": "ready"
  }
}
GET
/checkAdkStatus

Google ADK 모델의 상태를 확인합니다.

SDK 및 라이브러리

다양한 프로그래밍 언어를 위한 SDK를 제공합니다.

JavaScript/Node.js

npm install llm-quality-radar-sdk

const QualityRadar = require('llm-quality-radar-sdk');
const client = new QualityRadar('YOUR_API_KEY');

const result = await client.generateTest({
  url: 'https://example.com',
  scenarios: [/* ... */]
});

Python

pip install llm-quality-radar

from llm_quality_radar import QualityRadarClient

client = QualityRadarClient(api_key='YOUR_API_KEY')
result = client.generate_test(
    url='https://example.com',
    scenarios=[...]
)

사용 제한

API 남용을 방지하기 위해 다음과 같은 제한이 적용됩니다:

계정 유형 분당 요청 수 월별 테스트 수 동시 실행
Free 10 100 1
Pro 100 10,000 5
Enterprise 1,000 무제한 50

오류 코드

상태 코드 오류 유형 설명
400 Bad Request 잘못된 요청 형식 또는 누락된 필수 파라미터
401 Unauthorized 유효하지 않은 API 키
403 Forbidden 권한 없음 또는 사용 제한 초과
429 Too Many Requests 속도 제한 초과
500 Internal Server Error 서버 내부 오류

오류 응답 형식

{
  "error": "Validation Error",
  "message": "URL is required",
  "code": "MISSING_REQUIRED_FIELD",
  "timestamp": "2024-01-15T09:30:00.000Z"
}
